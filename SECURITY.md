# Security Policy

## Supported versions

Security fixes are applied on a best-effort basis to the latest state of the default branch.

| Version | Supported |
| --- | --- |
| `main` | ✅ |
| Older branches or tags | ❌ |

## Reporting a vulnerability

Please **do not open a public GitHub issue** for security vulnerabilities.

Preferred reporting path:

1. Use GitHub's **private vulnerability reporting** / security advisory flow for this repository if it is available to you.
2. If private reporting is unavailable, contact the maintainer through the repository owner's GitHub profile and clearly label the report as a security issue.
3. Include affected versions, reproduction steps, impact, and any suggested mitigations.

## What to expect

- Initial triage acknowledgment target: **within 5 business days**
- Status updates will be shared as the report is investigated
- We will coordinate disclosure timing with the reporter when a fix is required

## Scope notes

Relevant classes of reports for this repository include:

- Sensitive data exposure
- Unsafe network or parsing behavior
- Dependency vulnerabilities that materially affect shipped builds
- Privacy-impacting bugs involving location, camera, or user data handling

General support requests and feature ideas should go to [SUPPORT.md](SUPPORT.md) or the public issue tracker, not through the security channel.
