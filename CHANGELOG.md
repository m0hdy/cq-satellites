# Changelog

All notable project changes should be recorded in this file.

This project aims to follow the spirit of [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) for human-readable release notes, while GitHub Releases can publish generated notes for each tagged release.

## Release policy

- Public release tags should use a `vMAJOR.MINOR.PATCH` format such as `v1.2.0`
- Release automation may derive App Store build metadata from the CI run number
- Tagged releases are the public source of truth for shipped versions
- Generated GitHub release notes supplement, but do not replace, curated changelog updates for notable user-facing changes
